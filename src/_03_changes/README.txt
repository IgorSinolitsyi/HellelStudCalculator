1) Изменим строку информации о приложении.

src\_03_changes\master_01\Main.java
изменено на
src\_03_changes\master_02\Main.java

Цвет файла изменился.


2) Смотрим изменения 
IDE, нижнее меню
    Git>Вкладка Local Changes  

Есть информация о файле для коммита (хеш коммита, 
изменения в коде).


3) Произведем коммит изменений

Во вкладке Local Changes Правый клик на файле>Commit File...  

Откроется интерфейс коммитов. Внимание на изменения файла.

В интерфейсе коммитов, введем сообщение коммита 
Update app info

Жмем Commit.

В проекте произошли изменения.
Цвет файла(ов) под фиксацией изменится на цвет по умолчанию.
Вкладка Local Changes теперь пуста. 
Во вкладке Log увидим хронологию наших действий.


4) Запушим изменение в удаленный репозиторий.
IDE, верхнее меню
    Git>Push...

Появится окно, где указаны: коммит, файл для пуша, связка 
локального проекта/репозитория с удаленным репозиторием.

Жмем Push.

Если все хорошо, внизу-справа появится сообщение об успешном пуше.


5) Проверим удаленный репозиторий.
В браузере, перезагрузим страницу удаленного репозитория.
Можем увидеть когда произведен был коммит, количество коммитов. 


6) Добавим функционал сложения и проверим изменения.
IDE, нижнее меню
    Git>Вкладка Local Changes

Есть информация о файле для коммита (хеш коммита, изменения в коде).


7) Произведем коммит изменений (функционал сложения) с сообщением коммита
    Add addition

В проекте произошли изменения.
Проверим вкладки Local Changes и Log в панели Git.


8) Запушим изменения (функционал сложения) в удаленный репозиторий.
IDE, верхнее меню
    Git>Push...

Появится окно, где указаны: коммит, файл для пуша, связка 
локального проекта/репозитория с удаленным репозиторием.

Жмем Push.

Если все хорошо, внизу-справа появится сообщение об успешном пуше.


9) Проверим удаленный репозиторий.
Через браузер зайдем на страницу репозитория.
Можем увидеть когда произведен был коммит, количество коммитов.
Кликая на коммиты, можем увидеть состояние кода и изменения в нем 
на момент коммита.
