1) Создадим удаленный репозиторий нашего проекта.

Войдем в свой аккаунт на GitHub.

Для консистентности, имя удаленного репозитория будет 
аналогично проекту на локальной машине.

Можем сделать репозиторий открытым (public) или 
закрытым (private). Делаем public.

В опции Initialize this repository with: ничего не отмечаем.
Жмем "Create repository".

Отобразится первоначальная страница репозитория. 
Пока, интересует ссылка на этот репозиторий.
Она подобна
https://github.com/[автор]/[имя репозитория].git

Можем сохранить ее у себя на компьютере.


2) Свяжем локальный проект с репозиторием на 
GitHub (удаленный репозиторий).
IDE, верхнее меню    
    Git>Manage Remotes...

Появится окно выбора удаленного репозитория, где 
необходимо указать его имя и URL (расположение в сети).

Нажмем  +  вверху-слева.

В появившемся окне, оставим имя неизменным (как правило, origin), 
а в поле URL скопируем ссылку на наш GitHub репозиторий.

Жмем OK.

ВНИМАНИЕ. Может появиться запрос залогиниться.
    Жмем Log In via GitHub...
    В браузере может появится страница от JetBrains с 
    необходимостью авторизоваться в GitHub. 
    Авторизируемся.
    Далее страница Authorize JetBrains IDE Integration. 
    Авторизируемся.

В IDE, в окне выбора удаленного репозитория появятся имя и 
URL репозитория на GitHub.
Выделим и жмем OK.

ВНИМАНИЕ. Можем предварительно, через IDE, связать IDE со 
своим GitHub аккаунтом
    File>Settings>Version control>GitHub>+>
    Log In via GitHub...>Авторизируемся через страницу IDE на GitHub

    https://www.jetbrains.com/help/idea/github.html#register-account  


3) Отправим (запушим) проект в удаленный репозиторий на GitHub.
IDE, верхнее меню   
    Git>Push...
Появится окно, где указаны: наши коммиты, файл для пуша, связка 
локального проекта/репозитория с удаленным репозиторием.

Жмем Push.

Если все хорошо, появится сообщение об успешном пуше.


4) Проверим удаленный репозиторий на GitHub.
В браузере, перезагрузим страницу репозитория. 
Должна отображаться информация о репозитории, 
находиться папки и файл(лы) проекта.

Мы запушили проект на удаленный репозиторий :)

Кликая на пакеты и файлы репозитория, можем открывать их 
и перемещаться по репозиторию.
