1) (master) Создадим новую ветку - develop. 
У нас, пока, одна ветка  master, находимся в ней.

IDE, нижнее меню 
    master>master>New Branch from 'master'...

В появившемся окне введем название новой ветки  develop . 
Затем жмем Create.
Новая ветка создана. Находимся в ней. 
Внимание на нижнее меню.


2) (develop) Посмотрим наши коммиты. 
Перейдем во вкладку Log, где увидим хронологию наших действий.
Пока, никаких изменений.


3) (develop) Добавим функционал умножения и проверим изменения.

src\_04_branching\develop_01\Main.java
изменено на
src\_04_branching\develop_02\Main.java

IDE, нижнее меню
    Git>Вкладка Local Changes

Есть информация о файле для коммита (хеш коммита, изменения в коде).


4) (develop) Произведем коммит.
Сообщение коммита
    Add multiplication

IDE, верхнее меню
    Git>Commit...

В проекте произошли изменения.
Проверим вкладки Local Changes и Log в панели Git.


5) (develop) Сравним ветки.
IDE, нижнее меню
    develop>master>Compare with 'develop'

Откроется окно, где видимые различия между ветками 
(все коммиты, которые есть в develop, которых нет в master).


6) (develop) Перейдем в ветку master
IDE, нижнее меню 
    develop>master>Checkout


7) (master) Сравним ветки.

Перейдем во вкладку Log, где увидим хронологию 
наших действий. 
Внимание на выделение строк.


8) (master) Перейдем в ветку develop
IDE, нижнее меню
    master>develop>Checkout

Посмотрим на код.


9) (develop) Запушим изменения в удаленный репозиторий.
IDE, верхнее меню
    Git>Push...

Появится окно, где указаны: коммит, файл для пуша, связка 
локального проекта/репозитория с удаленным репозиторием.

Жмем Push.

Если все хорошо, внизу-справа появится сообщение об успешном пуше.


10) Проверим удаленный репозиторий.
Через браузер зайдем на страницу репозитория.
Можем увидеть когда произведен был коммит, количество коммитов.
Появилась дополнительная ветка.

Исследуем ветки, коммиты.
